## System Health Protocol — "Installed ≠ Working"

**The #1 mistake:** Installing a tool (MCP server, hook, memory layer) and assuming it works. Installing is 10%. WIRING it (telling Claude WHEN and HOW to call it) is 90%.

### The 3-Layer Health Check

| Layer | What to Check | How to Diagnose | Fix |
|-------|--------------|----------------|-----|
| **1. Installed?** | Is the MCP/hook/script in `settings.json` or `hooks/`? | `cat ~/.claude/settings.json \| grep "tool-name"` | Add it to settings.json |
| **2. Wired?** | Does CLAUDE.md tell Claude WHEN to call it? | Search CLAUDE.md for the tool name | Add usage rules to CLAUDE.md |
| **3. Firing?** | Has Claude actually CALLED it in a session? | Check enforcer tracking: `cat /tmp/claude-backup-track-*.json` | The enforcer nags until it fires |

### Common Failure Modes

| Symptom | Root Cause | Fix |
|---------|-----------|-----|
| "I installed 6 memory MCPs but none save" | Installed but not WIRED — CLAUDE.md has no rules for when to call them | Add 6-Layer Memory Stack table to CLAUDE.md |
| "Content ideas aren't being pushed" | Rule exists but says "after every build" — too vague | Make triggers explicit: "built a skill? Push idea NOW" |
| "Google Drive saves aren't happening" | Enforcer was matching `gdrive` but MCP tool is `google-drive` | Match EXACT MCP tool names in enforcer |
| "Hooks aren't firing" | Wrong event name or hook returns non-zero exit | Check `~/.claude/settings.json` hook config, test with `echo '{}' \| node hook.js` |
| "Supermemory runs out of credits" | Calling every session (30 credits/month) | Add daily cooldown (1x/day max) |
| "Session dies mid-build" | Context bloat — no size-gate or diet system | Install size-gate.js + skill-diet.py |
| "MCP tool not found" | Server crashed or not started | Restart Claude Code. Check `claude mcp list` |
| "Memory saves are slow / inconsistent / duplicating" | **Memory MCP bloat** — running 5+ memory servers simultaneously (mem0, memorix, omega-memory, memory-knowledge-graph, memory-service all active at once) | Pick ONE session memory (Engram) + ONE global recall (Supermemory). Remove the rest from your project config in `~/.claude.json`. More memory MCPs ≠ better memory. |
| "WhatsApp / TypeScript MCP won't start" | MCP config uses `node` to run a `.ts` file — Node can't execute TypeScript directly | Change command to `npx tsx` in `~/.claude.json`. Check if tsconfig has `noEmit: true` — if so, the project is tsx-native and was never meant to compile. |

### The Leila Audit Fix (Run This When Things Feel Broken)

```bash
# 1. Check what MCPs are installed
cat ~/.claude/settings.json | python3 -c "import sys,json; d=json.load(sys.stdin); print('\n'.join(d.get('mcpServers',{}).keys()))"

# 2. Check what hooks are installed
ls ~/.claude/hooks/

# 3. Check enforcer tracking (what's been called this session)
cat /tmp/claude-backup-track-*.json 2>/dev/null | python3 -m json.tool

# 4. Check if daily Leila Audit is running
launchctl list | grep leila

# 5. Check hook health (test each hook)
for f in ~/.claude/hooks/*.js; do echo "Testing $f..."; echo '{"tool_name":"test","tool_input":{},"session_id":"test"}' | node "$f" 2>&1 | head -1; done
```

### The Golden Rule

**Every tool needs 3 things to work:**
1. **Installation** — the tool exists in settings.json
2. **Wiring** — CLAUDE.md has explicit rules for WHEN and HOW to call it
3. **Enforcement** — a hook or protocol that NAGS until it's done

If ANY of these 3 are missing, the tool is dead weight. The Leila Principle: "What's already in the building that nobody's using?"

## Engineering Principles — DRY KISS YAGNI SOLID (Add to Your CLAUDE.md)

Every serious engineer follows these. Add to your `CLAUDE.md`:

```markdown
## DRY KISS YAGNI SOLID (Engineering Laws)
- **DRY** — Don't Repeat Yourself. One source of truth. If it exists, import it.
- **KISS** — Keep It Simple. Simplest solution that works = best solution.
- **YAGNI** — You Ain't Gonna Need It. Build what's needed NOW, not "just in case."
- **SOLID** — Single Responsibility (1 skill = 1 job). Extend, don't rewrite.
Over-engineering check: "Is this the simplest version? Did someone ask for this? Does this already exist?"
```

## Gate Protocol
- **Pre-flight:** Verify all dependencies and tools are available. Check file paths exist. Confirm no destructive operations without backup.
- **Mid-flight:** Log each operation. Verify intermediate outputs before proceeding to next step. Handle errors gracefully — never fail silently.
- **Post-flight:** Verify system state is correct. Confirm all outputs saved to expected locations. Test that the system works end-to-end.
