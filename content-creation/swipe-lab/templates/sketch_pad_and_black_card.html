Ok so 3 things make React re-render:

And #3 is the one that gets literally everyone.

1. State changes
You call useState → screen re-paints. Pretty straightforward.
2. Props change
Parent sends new data → child updates. Makes sense right?
3. Parent re-renders
THIS ONE THO. Parent changes → ALL children re-render. Even if literally nothing changed for them.

The analogy

One kid gets caught cheating → teacher makes the WHOLE CLASS redo the test. Everyone suffers because of one person. That's React without optimization.

The fix: React.memo()

Basically a "Do Not Disturb" sign for your components.

WITHOUT MEMO

App changes
↓
Header re-renders ❌
BigList re-renders ❌
Footer re-renders ❌

slow af

WITH MEMO ✨

App changes
↓
Header SKIPPED ✅
BigList updates (props changed)
Footer SKIPPED ✅

way faster

// Literally just wrap it. That's the whole thing.
const Header = React.memo(({ title }) => {
  return <h1>{title}</h1>
})
// Now it only re-renders when title actually changes
// React sees the "Do Not Disturb" sign and walks past
but don't go crazy with it → 4 / 6 @itisdrey

When to memo (and when to chill)

Don't put a Do Not Disturb sign on every door in the hotel. Just the ones where someone's sleeping.

✓ Yeah memo this

🔵 Big lists (100+ items)

🔵 Charts & data viz

🔵 Components that re-render a LOT but props barely change

🔵 You actually MEASURED it's slow

✗ Nah skip it

🔴 Simple buttons / text

🔴 Stuff that changes every render anyway

🔴 Tiny cheap components

🔴 "Just in case" (this is not a thing)

The reality is

Premature optimization is a waste of your time. If you can't PROVE it's slow with actual numbers, don't touch it. Measure first. Always.
